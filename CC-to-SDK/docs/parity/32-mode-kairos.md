| 32.4 | Channels: bidirectional MCP-over-channel inbound + permission relay | 🏗 build | — | The transport is MCP, which the SDK fully supports (mcpServers, SSE/HTTP/stdio). The harbor allowlist, the <channel> envelope, and the permission-reply protocol are CC conventions you reimplement; concrete adapter servers are external MCP plugins. | P2 | doc | feb |
| 32.5 | Cron / scheduled triggers (CronCreate/Delete/List + /loop) | 🏗 build | — | A pure scheduling/persistence concern with no LLM dependency — write a cron loop in your harness that, on fire, pushes a prompt into the query() generator (or starts a new query).
