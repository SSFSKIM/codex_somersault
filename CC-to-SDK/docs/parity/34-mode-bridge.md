# Parity — 34-mode-bridge

| id | feature | verdict | SDK surface | bridge / gap | phase | conf | snap |
|---|---|---|---|---|---|---|---|
| 34.1 | Remote Control: drive a local REPL from claude.ai/code (web/mobile) | 🚫 not-possible | — | The entire bridge talks to Anthropic-hosted endpoints (/v1/environments/bridge, /v1/code/sessions, session_ingress WS/SSE) authenticated with claude.ai OAuth/JWT — a claude.ai-internal relay. The SDK exposes no Remote Control client/server; you cannot recreate the claude.ai/code control plane. | Pnon-goal | doc | feb |
| 34.2 | Standalone remote-control server (claude remote-control / rc) | 🚫 not-possible | — | Its work-poll/spawn loop is bound to Anthropic's environments/code-sessions API and JWT ingress (claude.ai-internal). The generic idea of 'a server that spawns SDK sessions on demand' is buildable (see 33.1), but parity with the claude.ai-coupled Remote Control server is not. | Pnon-goal | doc | feb |
| 34.3 | Bridge control-request protocol (set_model/permission/interrupt/initialize) | 🏗 build | — | The control surface maps onto SDK Query methods (setModel, setPermissionMode, setMaxThinkingTokens, interrupt) in streaming mode. If you build your own transport you can translate inbound control frames to these calls — the methods are provided, but the bridge's claude.ai wire protocol/transport is not, so the glue is build. | P2 | doc | feb |
| 34.4 | Trusted-device enrollment + JWT session-ingress auth | 🚫 not-possible | — | Trusted-device enrollment and session-ingress JWTs are issued/validated by Anthropic's bridge backend (claude.ai-internal auth). The SDK has no hook into this auth flow; there is nothing to configure or rebuild client-side without the hosted endpoints. | Pnon-goal | doc | feb |
| 34.5 | BRIDGE_SAFE_COMMANDS allowlist (phone-safe slash commands) | 🏗 build | — | Slash-command availability is a harness UI policy. With the SDK you enumerate commands via query.supportedCommands() and gate which ones your remote UI exposes; the specific safe-list is a convention you reimplement. Not meaningful without a remote UI (which itself is not-possible/34.1). | P3 | inferred | feb |
