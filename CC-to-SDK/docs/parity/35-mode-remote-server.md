# Parity — 35-mode-remote-server

| id | feature | verdict | SDK surface | bridge / gap | phase | conf | snap |
|---|---|---|---|---|---|---|---|
| 35.1 | CCR — agent runs in Anthropic-managed cloud container | 🚫 not-possible | — | CCR containers, the /v1/sessions/ws subscribe channel, and the GitHub import-token / environment-provider endpoints are all Anthropic-hosted (claude.ai-internal). The SDK cannot provision or attach to a CCR container. Run your own cloud sandbox with the SDK if you need remote execution — that is a separate build, not CCR parity. | Pnon-goal | doc | feb |
| 35.2 | Direct-connect server + cc:// client (claude server / open) | 🏗 build | DirectConnectTransport, parseDirectConnectUrl, DirectConnectError (runtime-exported) | The SDK ships the CLIENT side: DirectConnectTransport + parseDirectConnectUrl + DirectConnectError are exported, so connecting to a cc:// server is configurable. The SERVER (`claude server` host) is not an SDK API, so hosting the session is build (or use the CLI's own `claude server`). Client connect ≈ provided; the host runtime ≈ build. | P2 | doc | feb |
| 35.3 | SSH remote execution (claude ssh <host>) | 🏗 build | — | No SSH-remote primitive in the SDK. spawnClaudeCodeProcess lets you launch the underlying CC process over a custom transport (e.g. an SSH-tunneled command), which is the closest hook; the deploy/reverse-auth-proxy plumbing is entirely yours to build. | P2 | inferred | feb |
| 35.4 | Self-hosted runner (claude self-hosted-runner) | 🚫 not-possible | — | The runner polls an Anthropic worker-service backend (claude.ai-internal); the SDK exposes no runner-registration/poll API. A bespoke job-runner that drives query() is buildable but is not parity with the hosted self-hosted-runner protocol. | Pnon-goal | inferred | feb |
| 35.5 | UDS inbox / peers (local Unix-domain-socket message bus) | 🏗 build | — | A local IPC bus between agent processes is harness-level. Build it with a UDS server of your own and feed inbound messages into each session via streamInput; the SDK provides no peer/messaging-socket transport (SendMessage agent-team messaging is a CC tool, not an SDK API). | P2 | inferred | feb |
| 35.6 | /ultrareview (Teleported) — remote CCR review session | 🚫 not-possible | — | It launches a CCR (cloud) session (35.1) and inherits the same claude.ai-internal dependency. A local equivalent (run a review prompt via query()) is trivially buildable, but the Teleported/remote-execution behavior is out of reach. | Pnon-goal | inferred | feb |
| 35.7 | CLAUDE_CODE_REMOTE container-side behavior reshaping | 🚫 not-possible | — | These are internal behavior flips inside the CC binary when it runs as a CCR container guest; the variable is set by the CCR controller out-of-band and is meaningful only in that hosted context. The SDK does not run inside CCR and exposes none of these toggles individually. | Pnon-goal | doc | feb |
